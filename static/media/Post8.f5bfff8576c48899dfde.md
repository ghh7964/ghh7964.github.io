근데 이제 저는 두 가지 스펠을 가지고 있죠.  
바로 2마리의 피를 1씩 깎는 광역기랑 ~~AD 베이가 Q~~

자 그러면 이 문제를 해결하려고 하는 입장에서  
어떤 상황에서 단일기를 사용하고, 또 어떤 상황에서 광역기를 사용하는 것이 비용을 적게 소모하는지를 분석해야 합니다.

광역기는 어차피 2마리한테만 사용하니까 값을 한 쌍씩 잡아서 단일기와 광역기를 비교하면 되겠죠?
  
  
자 먼저,  
[1, 1]은  
단일기: [1, 1] -> [0, 1] -> [0, 0] = 2  
광역기: [1, 1] -> [0, 0] = 1  
이니까 당연히 광역기를 사용하는 게 비용을 적게 소모하죠?  
광역기를 사용하면 됩니다!
  
[1, 2]는  
단일기: [1, 2] -> [0, 2] -> [0, 0] = 2  
광역기: [1, 2] -> [0, 1] -> [0, 0] = 2  
이니까 비용이 똑같죠? 아무거나 사용해도 상관없습니다.  

[2, 2]는  
단일기: [2, 2] -> [0, 2] -> [0, 0] = 2  
광역기: [2, 2] -> [1, 1] -> [0, 0] = 2  
이니까 이 경우에도 비용이 같네요.

B번입니다.

또 대충 문제 해석을 해보자면  
Eve라는 사람이 Alice와 Bob을 웃겨야 하는데  
Eve가 칠 수 있는 드립의 종류가 4가지가 있대요

1. 둘 다 좋아함, 2. Alice는 좋아하고 Bob은 싫어함, 3. Alice는 싫어하고 Bob은 좋아함, 4. 둘 다 싫어함

라는데 드립이 좋으면 mood가 1 증가하고 싫으면 mood가 1 감소하는 특이한 관객들입니다  
근데 이제 관객 둘 중 한 명이라도 mood가 -1이 되면 관객이 떠나고 쇼를 끝낸다고 해요  
아니면 Eve가 칠 드립이 없어져도 쇼를 끝냅니다.  

그래서 Eve가 쇼를 끝내지 않고 칠 수 있는 최대 드립 시전 횟수를 구하면 된다고 하네요.  
  
  
이건 A번에 비하면 좀 복잡하니까 입력 예시와 출력 예시를 살펴봅시다!

5 0 0 0  
이건 1번째 드립만 있으니까 관객이 떠나지 않고, 칠 드립 없어져서 쇼가 끝난 경우이므로  
```Output: 5```입니다. 간단하죠?

0 0 0 5  
이건 4번째 드립만 있으니까 그냥 드립 한 번 치고 관객 둘 다 mood -1 되면서 쇼가 끝난 거죠?  
```Output: 1```입니다. 이것도 간단해요.

2 5 10 6  
이건 보기만 해도 전에 비해 좀 복잡해 보이죠?  
그래도 차근차근 분석해 보자고요!  
일단 1번째 드립을 2번 소모하면  
[0 5 10 6]에 관객 상태는 (2, 2)가 되겠죠?  
근데 여기서 바로 4번째 드립을 소모할 수는 없으니까,  
2번째와 3번째 드립을 적절히 사용하여 줄을 타줘야 합니다.  
[0 4 10 6] (3, 1) -> [0 5 9 6] (2, 2)  
이런 식으로 쭉 반복해 줘요  
[0 0 5 6] (2, 2)  
그러면 이렇게 2번째 드립이 다 소모되는데,  
이 상황에는 결국 3번째 드립, 4번째 드립 상관없이  
뭘 사용하더라도 mood -1이 되면서 쇼가 끝납니다.

사실상 이 예시에서 패턴을 전부 파악할 수 있어요.  
일단 1번째 드립을 전부 소모하면 ```count: 2```,  
2번째 드립과 3번째 드립으로 티키타카를 진행하는데  
이 횟수는 2번째 드립과 3번째 드립 중 더 작은 값의 2배입니다.(티키타카니까)  
```count : 2 + (5 * 2) = 12```,  
그리고 (관객 상태 + 1)을 소모해야 합니다.  
그런데 어차피 관객 상태는 티키타카가 진행되면서 바뀌지 않고  
1번째 드립을 소모했을 때 이미 결정되므로  
```count : 12 + (2 + 1) = 15```,  
```Output: 15```입니다. 식을 일반화해볼까요?

드립의 종류 순서대로 변수명을 ```a, b, c, d```라고 선언하고  
2번째 드립과 3번째 드립 중 값이 큰 것을 ```foo```, 작은 것을 ```bar```라고 할 때,  
```Output = a + (2 * bar) + (a + 1) = 2 * (a + bar) + 1 ```입니다.  

근데 방금 일반화한 이 식은 특정 조건을 만족해야 합니다.  
Eve가 칠 드립이 충분해야 한다는 거죠.  
관객 중 한 명의 mood가 -1이 되기 전에 쇼가 끝나버리면  
마지막에 (관객 상태 + 1)을 카운트하는 것에 문제가 생기기 때문입니다.

그래서 드립이 부족하여 쇼가 끝난다고 가정한 뒤 식을 다시 세워보면,  
```a + 1```을 더하는 것이 아닌, ```(foo - bar) + d```를 더해야 합니다.  
즉, ```Output = a + (2 * bar) + min(a + 1, foo - bar + d)```이 최종 식인 거죠.  
(단, ```min()``` 함수는 더 작은 값을 리턴한다)

C번입니다.

이건 처음에 문제 해석할 때 뭔 소리인지 모르겠어서 좀 당황했습니다.  
그냥 대충 순열을 하나 던져주는데, 그걸 특정 작업을 수행하여 오름차순 정렬하면 됩니다.  
그 작업이 뭐냐면 한 쌍의 값을 잡고 둘 중 작은 값을 수열의 가장 왼쪽으로, 큰 값을 가장 오른쪽으로 보내는 겁니다.  
그리고 수열을 정렬하는 최소 작업 횟수를 구하면 돼요.  
  
  
일단 예시를 통해 파악을 좀 해봅시다!

[1, 5, 4, 2, 3]  
=> (2, 4), [2, 1, 5, 3, 4]  
=> (1, 5), [1, 2, 3, 4, 5]  
```Output: 2```입니다.  
이게 무지성으로 수를 고른 것 같아도 여러 번 예시들을 분석해 보면,  
특정 쌍을 순서대로 선택해야 최소 작업 횟수를 만족할 수 있더라고요.  
가령 위의 테스트케이스는 n이 5이므로  
[1, 2, 3, 4, 5], (2, 4), (1, 5)  
3을 중심으로 하여 (2, 4)와 (1, 5)처럼  
양쪽으로 확산되는 숫자 쌍대로 작업을 진행해야 합니다.  
머리 좀 굴려보면 당연한 거라, 추가 설명을 하지는 않겠습니다.  

[5, 2, 4, 1, 6, 3]  
=> (3, 4), [3, 5, 2, 1, 6, 4]  
=> (2, 5), [2, 3, 1, 6, 4, 5]  
=> (1, 6), [1, 2, 3, 4, 5, 6]  
```Output: 3```입니다.  
짝수의 경우는 어떻죠?  
그냥 중앙에 있는 숫자 쌍을 처음 선택하고 다음 숫자 쌍은  
처음 숫자 쌍에서 점차 양쪽으로 확산됩니다. 비슷하죠?  
그래서 순열의 길이인 n을 활용하여 숫자 쌍을 일반화해보자면,  
```if(n % 2 == 1) (n / 2, (n / 2) + 2) ```  
```else (n / 2, (n / 2) + 1)```가 됩니다.  
그리고 ```while```문을 통하여 ```(n - 1, n + 1)```을 계속 진행하면서  
다음 숫자 쌍을 찾아주면 됩니다.

[2, 1, 4, 3]  
=> (1, 4), [1, 2, 3, 4]  
```Output: 1```입니다.  
그런데 문제는 "최소" 작업 횟수죠?  
방금까지 분석한 대로는 위의 순열에서 (2, 3)을 작업해야 하지만,  
굳이 (2, 3)을 거치지 않고 바로 (1, 4)를 작업하더라도 문제없이 정렬이 됩니다.  
이유는 간단합니다. (2, 3)이 이미 순열에서 순서대로 놓여있기 때문이죠.  
그래서 굳이 작업을 거칠 필요가 없는 겁니다. (1, 4)가 알아서 밖으로 빠져주니까요

[1, 6, 2, 3, 4, 5]  
=> (1, 6), [1, 2, 3, 4, 5, 6]  
```Output: 1```입니다.  
이 경우에도 비슷합니다.  
(1, 6) 작업 전에 (3, 4)와 (2, 5)의 작업이 진행되어야 하지만,  
두 작업이 스킵됐죠? 왜냐하면 (3, 4)가 이미 순서대로 놓여있고  
순열 내 (3, 4) 양쪽에 (2, 5)가 각각 놓여있기 때문입니다.

그렇다면 어떻게 알고리즘을 설계해야 할까요?  
먼저, 앞서 설명했던 것처럼  
```if(n % 2 == 1) (n / 2, (n / 2) + 2) ```  
```else (n / 2, (n / 2) + 1)```를 통해 첫 번째로 작업되는 숫자 쌍을 찾아야 하고  
그 숫자 쌍이 주어진 순열 내에서 순서대로 배열되어 있는지를 점검해야 합니다.  
만약 순서대로 배열되어 있지 않다면, 모든 숫자 쌍의 작업이 진행되어야 하므로  
```Output = n / 2```가 나오겠죠? 숫자 "쌍"이니까 당연합니다.

만약 순서대로 배열되어 있다면 일단 ```count++```를 하고  
[1, 6, 2, 3, 4, 5]와 같은 테스트케이스가 존재하기 때문에  
(3, 4)에서 3의 왼쪽에 다음 숫자 쌍인 (2, 5)의 2가 존재하는지,  
4의 오른쪽에 5가 존재하는지를 찾고 만약 둘 다 존재한다면  
```count++``` 해주시고 이를 반복하면 됩니다.  
다음에 나올 숫자 쌍을 찾는 법은 제가 아까 잠깐 설명했듯이  
```while```문과 ```for```문을 적절히 활용해서 순차 탐색하면 되겠죠?  
그러면 최종적으로 ```Output = (n / 2) - count```가 됩니다.  

[1, 2, 4, 3, 5]  
=> (2, 4), [2, 1, 3, 5, 4]  
=> (1, 5), [1, 2, 3, 4, 5]  
아 근데 홀수인 경우엔 조건이 하나 더 필요합니다.  
첫 번째 숫자 쌍이 순서대로 배열되어 있다고 하더라도,  
위와 같은 테스트케이스에선 보이는 것처럼 소용이 없죠?  
3이 (2, 4) 바깥쪽에 존재해서 그렇습니다.  
그래서 ```n % 2 == 1```이라면 주어진 순열에서  
```(n / 2, (n / 2) + 2)``` 사이에 ```(n / 2) + 1```의 존재 유무를  
확인하고 다음 숫자 쌍을 탐색하면 진짜 끝입니다!!!!
